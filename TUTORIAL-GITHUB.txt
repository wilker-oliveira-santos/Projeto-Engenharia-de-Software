1. Escolha uma ferramenta de GCS. Para esta ferramenta:

	A ferramenta escolhida foi o GitHub.


2. Descreva como colocar um artefato sob gerenciamento 
de configuração e então faça-o com algum artefato seu.

	Inicialmente o desenvolvedor deve criar uma conta no GitHub, caso o mesmo não tenha.
	Logo em seguida, o desenvolvedor deve criar um repositório(Público ou Privado).
	Depois, adicionar o(s) arquivo(s) ou por linha de código por um  editor, ou por via upload

	LINHA DE CÓDIGO

		- git init = inicializa um repositório.
		- git add <arquivo> = prepara o(s) arquivo(os) para um futuro commit.
		- git commit -m "Mensagem do desenvolvedor" = gera uma mensagem para o seu commit
		- git push = envia seus commits para um repositório.

	VIA UPLOAD

		- crie o repositório.
		- abra seu gerenciador de arquivos.
		- arraste o(s) arquivo(s) e envie para o repositório.


3. Descreva como é feita a autorização de um check-out.

	A autorização de check-out no GitHub/Git é utilizada para trocar ou baixar uma versão do arquivo, essa autorização pode ser efetuada de duas maneiras:

		Local - de qualquer modo todo usuário pode alterar os arquivos através de sua máquina.
		Remota - Só usuários autorizados(no repositório) podem fazer a alteração no GitHub.

	Ao registrar o Artefato no GitHub, o mesmo mostra o commit que o desenvolvedor fez e quem fez a alteração.
